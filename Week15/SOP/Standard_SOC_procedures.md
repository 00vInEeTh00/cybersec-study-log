## Introduction

A Security Operations Center (SOC) uses a tiered operating model (L1,
L2, L3) to monitor, detect, investigate, and respond to cybersecurity
incidents efficiently.

The tiered structure ensures alerts are handled according to complexity
and analyst expertise, improving response time and reducing security
risks.

------------------------------------------------------------------------

# Tier 1: Event Classification and Triage

Event classification and triage are critical components of a Security
Operations Center (SOC) incident handling process.

## Purpose

To identify, classify, and prioritize alerts based on risk level and
potential impact to the organization.

## Event Classification Process

SOC analysts (Tier 1) use a classification scheme to determine the
severity of each event or alert. This allows them to prioritize response
efforts based on organizational impact.

At present in our SOC, we analyze hosts and classify them into four
categories according to risk level.

### Host Risk Categorization

#### 1. Critical User (Risk Point = 100)

-   Users with risk score of 100.
-   Categorized as Critical.
-   Analysts observe installed software, user files, and system
    activity.
-   Requires immediate monitoring and escalation.

#### 2. High Risk Level User (Risk Point 70--99)

-   Represents a critical threat to the organization.
-   Requires immediate attention.
-   Examples: Data breach indicators, ransomware activity, privilege
    escalation attempts.

#### 3. Medium Risk Level User (Risk Point 30--69)

-   Significant events but not as critical as high-risk cases.
-   Examples: Phishing attempt, attempted exploit of known
    vulnerability.
-   Requires investigation and monitoring.

#### 4. Low Risk Level User (Risk Point 0--29)

-   Potential threat but low urgency.
-   Examples: Failed login attempt, low-level network scan.
-   Requires logging and observation.

# Tier 2: Prioritization and Analysis

Tier 2 analysts perform deeper investigation of escalated incidents from
Tier 1.

## Responsibilities

-   Review trouble tickets generated by Tier 1 analysts.
-   Utilize emerging threat intelligence (IOCs, updated detection
    rules).
-   Identify affected systems and determine the scope of attack.
-   Review and collect asset data (system configurations, running
    processes, network activity).
-   Determine and direct remediation and recovery efforts.

## Objective

Ensure the security team focuses on the most critical threats first and
responds in a timely and effective manner.

------------------------------------------------------------------------

# Tier 3: Remediation and Recovery

Tier 3 handles advanced remediation, system restoration, and long-term
security improvements.

## Responsibilities

-   Re-image systems and restore backups.
-   Patch or update systems (applications and operating systems).
-   Re-configure system access (account removals, password resets).
-   Re-configure network access (ACL updates, firewall rule changes, VPN
    access adjustments).
-   Review monitoring capabilities on servers and other assets.
-   Validate patching procedures and other security controls by running
    network vulnerability scans.

## Objective

Ensure systems are fully restored, secured, and hardened to prevent
recurrence of the incident.
